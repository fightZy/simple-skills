# Topic Summary Template

Use for topic-level aggregation under `summaries/topics/*.md`.

Metadata should follow [../schema.md](../schema.md).
Prefer scripts to generate frontmatter when topic-summary generation is implemented.

```md
# Topic Summary: workspace memory

## Current State
- What is currently true

## Key Decisions
- Important decisions to remember

## Relevant Crystals
- Crystal ids or links worth reading first

## Source Trail
- Source ids or filenames
```

# Session Template

Use for first-party session memory under `sessions/YYYY/MM/*.md`.

Metadata for this file should normally be generated by `scripts/record_session.py`.
Read [../schema.md](../schema.md) if you need the field rules.

```md
# Session: workspace memory skill setup

## Goal
What this session tried to resolve.

## Key Decisions
- Decision

## Rationale
- Why the decision was made

## Changes
- What changed in code, docs, or plans

## Open Questions
- What remains unresolved

## Follow-up
- Next actions

## Crystallization Candidates
- Rules, patterns, decisions, or insights worth promoting
```

# Advanced Brainstorming

本文档面向这个 skill 的维护者，不用于运行时路由。运行时入口仍然是 `SKILL.md`。

## 这个 Skill 是什么

`advanced-brainstorming` 是一个独立的高阶脑暴 skill，适用于用户希望把提案推离保守默认解、打开更大思路空间的场景。

它的目标是让 agent：

- 先显化隐藏假设，而不是照单全收用户的原始 framing
- 先顺着当前方向做增强，再主动做推翻和改写
- 产出能打破框架的替代方向和 wildcard
- 最后只做轻度收束，而不是直接进入实施规划

## 用途

当用户希望获得：

- 更强的头脑风暴
- 更不保守的思路拓展
- 更有想象力但仍可理解的方向
- 基于现有提案继续扩展出多条更强可能性

时，使用这个 skill。

这个 skill 不负责验证、调研或执行。

## 设计原则

- 用户最初的 framing 只是起点，不是边界。
- 好的回答应先扩展可能性空间，再轻度收束。
- 新意应来自关键轴的变化，而不是随机包装。
- wild idea 只有在仍然可解释时才有价值。
- 运行时回答应停在轻度优先级判断，不进入实施细节。

## 输出风格

运行时默认采用这样的结构：

1. hidden assumptions
2. expanded directions
3. frame-breaking alternatives
4. wildcards
5. most worth exploring next

之所以固定成这个结构，是因为基线脑暴虽然偶尔能给出有趣方向，但稳定性不够；这个 skill 的价值在于把更强的发散模式固化下来。

## 与其他 Skill 的关系

这个 skill 被有意设计为独立能力，不依赖仓库里的其他 skill，单独使用即可成立。

## 验证

常见验证方式应包含几类 prompt-pressure 场景：

- 一个本来就还不错、容易诱导 agent 直接做优化的提案
- 一个容易诱导 agent 输出空泛点子列表的模糊方向
- 一个明确要求“不要最稳妥方案”的请求

好的输出应体现：

- 明确的隐藏假设提炼
- 至少一段真正改变原始 framing 的扩展
- 分层的点子组，而不是单一平铺列表
- 有轻收束，但没有滑进执行规划
